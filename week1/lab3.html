Question 3

Google

The difference in the approach lays in two parts: Selecting the element and adding the action listener. The syntax for both is way shorter in jquery, but in that it is also lest volatile, abstracting away more of the details of how it works. While in the vanilla js version we explicitely state that we select by id, in jquery we use the # as a symbol for this (this would also be possible in pure js with the querySelector function. And instead of explicitely stating that we add an action listener for a certain action ('click'), we just give a function to what should happen on a click in jquery.

I dont think one is neccessarily better than the other, because while the vanilla js solution is longer it also yields more understanding about what is happening, and it is not too bad to read. The true difference herein for me lies in the importing of a huge 3rd party library for and easy task, which has load time, security and other implications. It also means that the name space for functions and variables is more flooded, which when including many 3rd party libraries can lead to many hard to debug problems.

In general the tendency of the industry is to go away from jquery, as javascript has evolved to a point where it is easy enough to use that we dont need it anymore. I personally consider that a good thing.

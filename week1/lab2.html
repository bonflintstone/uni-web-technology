Question 5

The W3C internationalization working group tackes the problem iof internationalization from a few different agles: A big one is the contribution to the unicode project which is working on creating a common character system which aims to make all characters from every language, dead or alive, displayable on every computer. Another one is the implementation of IDN, an effort to translate non ASCII domain names into ASCII ones which can be looked up by DNS. This makes people which are not familiar with the latin alphabet able to use domains in their own language, but it needs a client side application and is not implemented in the DNS system itself. The IETF also works on making text of all languages displayable in html and css, which means inlcuding support for different text orientations (ltr, rtl, top down) and displaying characters which change depending on the surrounding characters.
